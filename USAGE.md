# Nothing 使用说明

## 安装

1. 把 `nothing_V1.5.0.zip` 解压到一个文件夹
2. Chrome 地址栏输入 `chrome://extensions/`，打开"开发者模式"
3. 点"加载已解压的扩展程序"，选解压出来的文件夹
4. 扩展栏出现 N 图标就是装好了

## 基本用法

打开任意网页，点扩展栏的 N 图标。Nothing 会自动扫描当前页面并展示结果。

不需要手动触发扫描，打开即扫。

## 界面说明

弹出窗口有几个 Tab：

### 扫描（主界面）

最常用的页面。展示三种危险等级：

- **HIGH（高危）**：凭据密钥、JWT、XSS 漏洞等
- **MEDIUM（中危）**：内网 IP、邮箱、手机号、路径
- **LOW（低危）**：API 端点、URL、前端路由、JS 文件、文档文件等

点每个等级行可以展开看详情。每个分类旁边有"复制全部"按钮，一键复制该分类下所有数据。

### 漏洞

漏洞分会自动出现在扫描页的对应危险等级里。展开后能看到：

- 漏洞类型、置信度
- 分析过程（检测器是怎么判断的）
- 污点源 → 危险点的数据流
- POC 列表，可以直接点复制按钮复制 URL 到浏览器测试

如果开启了 POC 主动验证，标记"✓ 已验证弹窗"的就是实测能打的。

### 备份文件

扫描目标站点是否存在敏感文件泄露（.git、.env、备份压缩包等）。

需要先在设置里打开"备份文件扫描"。扫描可能需要几十秒，取决于站点路径数量。

### 指纹

展示当前站点识别到的技术栈信息——服务器、框架、CDN、安全头等。

### 命中日志

规则引擎命中记录。每次主动探测匹配到规则时，这里会记录一条。

### 排除

不想让某些 URL 出现在结果里，可以在这里加排除规则。支持通配符。

### 设置

- **主动扫描**：开启后自动探测规则引擎里配置的路径
- **备份文件扫描**：扫描敏感文件泄露
- **动态扫描**：监听 SPA 页面动态加载的 DOM 变化
- **深度扫描**：抓取页面引用的外部 JS 文件内容
- **AI 分析**：用 AI 过滤误报 + 分析漏洞（需要配 API Key）
- **POC 主动验证**：实测每条 XSS POC 是否真的弹窗（默认开）

AI 配置支持 OpenAI、Anthropic、DeepSeek、智谱 GLM、以及任意 OpenAI 兼容的自定义服务商。填好 API Key 和模型名，点保存就行。

也支持 JSON 导入——把含 API Key 的 JSON 配置文件粘贴进去，自动解析填充。

## 主动探测（规则引擎）

这是 Nothing 区别于纯被动扫描工具的核心能力之一。

### 什么是主动探测

光标移到某个页面上时，Nothing 会根据你配置的规则，对当前域名发起额外的 HTTP 请求，探测指定路径是否存在已知漏洞组件（比如 `/actuator/health`、`/xxl-job-admin/toLogin` 等）。

### 怎么加规则

两种方式：

**方式一：手动添加**

在选项页（右键 N 图标 → 选项，或 popup 里切到设置页）的规则编辑器里手动填路径、请求方法、匹配内容。

**方式二：指纹导入**

如果你有 xray/afrog 风格的 CEL 表达式指纹（比如别人分享的 shiro、xxl-job 检测规则），在 popup 的"指纹格式导入"弹窗里直接粘贴，工具会自动解析并转成规则。导入前可以预览解析结果。

整站指纹（像 shiro 这种整个框架级的）：如果配置了"整站指纹/同域名只报一次"，首次命中后同域名不会再重复告警，避免每个子路径都弹一次。

## 排除项

如果某个站点的特定路径每次扫出来一堆噪点，可以加排除规则：

- URL 排除：通配符匹配，命中后该 URL 不出现在结果里
- 分类排除：比如某站点确实没有 API 要关注，可以排除整个 API 分类

## XSS 检测和 POC 验证

这是 V1.3.0 之后的核心功能。

简单说：Nothing 会检查当前页面的 URL 参数有没有被反射到页面代码里，如果有且可以被利用执行 JS，就报一个 XSS 漏洞，并给出可以直接复现的 POC URL。

**POC 能直接用吗？**

默认会选一条最可能打通的做为主推（用 unicode 混淆关键字 + tagged template 绕过 WAF）。如果开了 POC 主动验证，就会在后台真的加载一下试试——弹了才标记"已验证"。

验证不会在页面上真弹框（hook 住了），也不会影响你正常浏览（后台标签静默跑）。

**POC 打不开/没弹窗怎么办？**

1. 看 POC 列表里有没有标记"已验证弹窗"的，有就用那条
2. 如果全部没验证通过（"POC 已验证"没出现），手动复制几条试试，可能目标有特殊的过滤
3. 可能目标确实不存在 XSS（误报），可以看看分析过程里的上下文判断依据

## 常见问题

**Q: 打开 popup 啥都没有？**
A: 切到目标网页再点 N 图标。popup 显示的是当前标签的扫描结果。

**Q: 高危漏洞数对不上？**
A: 徽标上的数字是 AI 分析和验证完成后的最终数。刚打开页面时还在分析中，等几秒会更新。

**Q: AI 分析一直显示"分析中"？**
A: 检查设置里的 API Key 和模型名是否正确。如果用的 DeepSeek 或智谱，确认端点地址没问题。

**Q: 插件不扫描了？**
A: Service Worker 可能被 Chrome 回收了。刷新目标页面一般就能恢复。如果频繁出现，可能是页面太"安静"（纯静态无网络请求），在设置里关掉"动态扫描"试试。

**Q: POC 验证显示"已验证"但没有弹窗效果？**
A: 验证是在后台标签跑的，你手动复制后在浏览器打开时需要**原样打开**（不能在新标签直接粘贴，因为有些 WAF 会检查 Referer）。正确做法：复制 POC URL → 新标签 → 地址栏粘贴回车。
